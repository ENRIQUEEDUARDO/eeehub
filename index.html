1. Introduction

EEEHUB is a comprehensive application designed to provide insights and management tools for e-commerce operations. It includes features for data import, inventory management, customer service tracking, and AI-driven analytics.

2. Installation

Prerequisites

Before installing EEEHUB, ensure you have the following installed on your system:

  • Node.js (LTS version recommended)
  • npm or yarn package manager
  • Git

Getting the Code

Clone the EEEHUB repository from its source:

git clone <repository_url>
cd eeehub

Installing Dependencies

Navigate to the project directory and install the necessary dependencies:

npm install
# or
yarn install

3. Setup

Environment Variables

Create a .env.local file in the root of the project. This file will contain your environment-specific configuration. Refer to a .env.example file (if provided) for the required variables. Common variables might include API keys, database URLs, etc.

# Example .env.local
DATABASE_URL="your_database_url"
API_KEY="your_api_key"
NEXT_PUBLIC_ANALYTICS_ID="your_analytics_id"

Database Setup (If applicable)

If the application requires a database, follow the specific database setup instructions (e.g., running migrations, seeding initial data). Consult any documentation or scripts provided for database setup.

API Key Configuration (If applicable)

Some features might require external API keys. Ensure these are correctly configured in your .env.local file as per the requirements of each feature.

4. Running the Application

Development Mode

To run the application in development mode with hot-reloading:

npm run dev
# or
yarn dev

The application will typically be accessible at http://localhost:3000.

Production Mode

To build the application for production and run it:

npm run build
# or
yarn build

npm start
# or
yarn start

5. Core Features

EEEHUB offers the following core features:

  • Dashboard: Provides an overview of key e-commerce metrics and visualizations.
  • Data Import: Allows importing sales, inventory, and customer data from various sources (e.g., CSV, APIs).
  • Inventory Management: Tools to track and manage product inventory levels.
  • Customer Service: Features to log and track customer issues and interactions.
  • AI Insights: Generates data-driven insights using AI models.

6. How to Use Each Feature

Using the Dashboard

The Dashboard is the first page you see upon logging in (if applicable). It displays charts and summary statistics related to your e-commerce performance. Use the filter controls (if available) to refine the data displayed.

Importing Data

Navigate to the "Data Import" section. You will likely find options to:

  • Upload CSV files: Click the "CSV Upload" button and select your data file. Ensure your CSV file is formatted correctly according to the application's requirements (refer to any provided templates).
  • Connect to APIs: If API connections are available, follow the instructions to configure your API credentials and select the data you wish to import.
  • Connect to Cloud Storage (e.g., Google Drive): Follow the authentication steps to connect your cloud storage account and select files for import.

Managing Inventory

The "Inventory" section allows you to:

  • View Inventory Summary: See a summary of your current stock levels.
  • Update Stock: Manually adjust inventory quantities (if the feature is available).
  • Track Product Movement: Monitor the flow of products (if detailed tracking is implemented).

Handling Customer Service Issues

In the "Customer Service" section, you can:

  • View Top Issues: See a chart or list of frequently occurring customer problems.
  • Log New Issues: Add details of new customer service interactions and issues.
  • Track Issue Status: Update the status of ongoing customer service cases.

Generating AI Insights

Navigate to the "AI Insights" section. Here, you can typically:

  • Select Data for Analysis: Choose the dataset or parameters for the AI model to analyze.
  • Generate Insights: Click a button to initiate the AI analysis.
  • View Insights: The generated insights will be displayed, often in the form of summaries, recommendations, or predictions.

7. Troubleshooting

If you encounter issues, consider the following:

  • Check the Console: Look for error messages in your browser's developer console or the terminal where the application is running.
  • Review Environment Variables: Ensure all required environment variables are set correctly in .env.local.
  • Verify Dependencies: Make sure all project dependencies are installed correctly (npm install or yarn install).
  • Consult the Repository Issues: Check the project's issue tracker on the code repository for known bugs or similar problems.

8. System Architecture and Vision

This section outlines the long-term vision and proposed system architecture for EEEHUB, focusing on its capabilities as an ETL pipeline and centralized e-commerce management tool.

8.1. Data Storage & Backend Processing (Firebase + Python Functions)

Firestore Collections:

  • products Collection:
    • sku (string, Document ID or unique field): Primary identifier.
    • title (string)
    • bodyHtml (string): Product description.
    • vendor (string)
    • productCategory (string)
    • type (string)
    • tags (array of strings)
    • published (boolean)
    • options (map/array of maps): e.g., [{name: "Color", value: "Red"}, {name: "Size", value: "M"}]
    • grams (number)
    • inventoryPolicy (string)
    • fulfillmentService (string)
    • requiresShipping (boolean)
    • taxable (boolean)
    • barcode (string)
    • imageSrc (array of strings/URLs): URLs to product images (ideally in Firebase Storage).
    • imageAltText (array of strings)
    • costPerItem (number)
    • priceMexico (number)
    • compareAtPriceMexico (number)
    • priceInternational (number)
    • compareAtPriceInternational (number)
    • status (string): e.g., 'active', 'archived', 'draft'.
    • metafields (map): For custom/generic data, e.g., metafields.googleShopping.customProduct, metafields.shopify.ageGroup.
    • reorderPoint (number): Threshold for reorder alerts.
    • forecastedDemand (number): Calculated by AI.
  • inventoryMovements Collection:
    • productId (string, reference to products document ID)
    • type (string: 'sale', 'receipt', 'adjustment', 'return', 'physical_count')
    • quantity (number, integer, can be negative for sales/adjustments)
    • timestamp (Firestore Timestamp)
    • source (string: 'Shopify Sale', 'Manual Adjustment', 'CSV Import', 'Private API Receipt', 'Meta Sale')
    • orderId (string, if applicable)
    • locationId (string, if multi-location)
    • reason (string, for adjustments/returns)
  • currentStock Collection (Optimized for fast reads):
    • productId (string, Document ID or unique field, maps to products.sku)
    • quantity (number, integer)
    • lastUpdated (Firestore Timestamp)
    • locationId (string, if multi-location)
  • salesData Collection (For richer Shopify/Meta insights):
    • orderId (string, Document ID or unique field)
    • sourcePlatformId (string, e.g., Shopify Order ID)
    • customerDetails (map)
    • lineItems (array of maps: productId, quantity, price, title)
    • totalPrice (number)
    • currency (string)
    • timestamp (Firestore Timestamp)
    • source (string: 'Shopify', 'Meta Shop', etc.)
    • fulfillmentStatus (string)
    • financialStatus (string)
  • insights Collection (Pre-calculated for dashboards):
    • insightId (string, e.g., daily_summary_YYYY-MM-DD, product_performance_SKU)
    • calculationDate (Firestore Timestamp)
    • data (map): Containing metrics like inventoryTurnover, daysOfSupply, bestSellingProducts, slowMovingItems, profitMargins, reorderAlerts, stockDiscrepancies.
    • Updated by scheduled Firebase Functions.
  • apiCredentials Collection (Secured - Not directly accessible by frontend):
    • provider (string: 'shopify', 'meta', 'private_api_X')
    • credentials (map): Encrypted API keys, tokens, etc.
    • userId (string): To associate credentials with a user/account.
    • Accessed only by secure Firebase Functions.

8.2. ETL Backend (Python Firebase Functions)

Python Firebase Functions will handle data extraction, transformation, and loading:

  • shopify_data_extractor Function:
    • Trigger: Cloud Scheduler (periodic) and Shopify Webhooks (real-time for orders, fulfillment).
    • Action: Connects to Shopify Admin API.
      • Pulls orders, updates inventoryMovements, currentStock, and salesData.
      • Pulls products (initial sync & updates), updates products collection.
      • Handles pagination and rate limits.
  • meta_insights_extractor Function:
    • Trigger: Cloud Scheduler (daily).
    • Action: Connects to Meta Graph API.
      • Fetches insights on product posts, engagement, clicks to shop.
      • Stores data in salesData (if direct sales) or a dedicated metaProductInsights collection linked to products.
  • private_api_extractor Function (for specific supplier/warehouse APIs):
    • Trigger: Cloud Scheduler.
    • Action: Connects to specified private APIs.
      • Parses inventory receipts or product master data.
      • Updates inventoryMovements, currentStock, and products.
  • csv_processor Function:
    • Trigger: Cloud Storage Trigger (on new CSV upload to a designated bucket).
    • Action: Uses pandas to read CSV.
      • Maps CSV columns to Firestore products fields, including custom metafields.
      • Handles product updates or creates new products based on SKU.
      • Processes Variant Inventory Qty into inventoryMovements and updates currentStock.
      • Image Handling: If Image Src in CSV are external URLs, downloads images, uploads them to Firebase Storage, and stores the Firebase Storage URL in products.imageSrc. Flags products needing images if Image Src is empty.
  • google_sheet_processor Function:
    • Trigger: Cloud Scheduler (hourly or on-demand).
    • Action: Uses gspread or Google API client to read a designated Google Sheet for manual adjustments (SKU, Quantity, Type, Reason).
      • Creates inventoryMovements and updates currentStock.
  • insights_calculator Function:
    • Trigger: Cloud Scheduler (e.g., daily or every few days).
    • Action: Calculates key metrics:
      • Inventory Turnover.
      • Days of Supply.
      • Best-Selling / Slow-Moving Items.
      • Reorder Alerts: Compares currentStock.quantity with products.reorderPoint and products.forecastedDemand / daysOfSupply.
      • Profit Margins per Product.
      • Stock Accuracy Discrepancies (requires baseline from physical_count movements or AI prediction).
    • Stores results in the insights collection.
  • ai_demand_forecaster Function (using Genkit):
    • Trigger: On-demand (e.g., from frontend for a specific product) or scheduled (for batch forecasting).
    • Action:
      • Takes product details (SKU, category, type, description, historical sales velocity if available).
      • Uses a Genkit flow to predict forecastedDemand.
      • Updates the forecastedDemand field in the products collection.
    • Can also be used to suggest an optimal reorderPoint.

8.3. Frontend Application (Next.js/React - Current Stack)

The existing Next.js application will be expanded to support these features:

  • Authentication: Firebase Auth.
  • Dashboard: Enhanced with data from the insights collection.
  • Product Management (within Inventory section):
    • Detailed product view/edit page showing all fields from the products collection (including custom metafields).
    • Forms to manually edit product details, including reorderPoint, costPerItem, prices.
    • Interface to upload/manage product images (linked to Firebase Storage).
    • Filtered view for products needing image uploads.
  • Data Ingestion Hub (Data Import section):
    • Secure forms for admin users to input API credentials (posts to a Firebase Function that saves to apiCredentials).
    • CSV upload component (already partially implemented).
    • Interface to trigger Google Sheet sync.
    • Buttons to manually trigger specific ETL functions (e.g., "Sync Shopify Now").
  • Reporting:
    • Downloadable CSV reports generated from the insights collection or direct queries.
  • Inventory Management:
    • Displaying current stock levels from currentStock.
    • Forms for manual stock adjustments (creating inventoryMovement records).
    • Displaying reorder alerts.

8.4. Key Features and User Benefits

  • Centralized Data: Aggregates data from multiple e-commerce platforms (Shopify, Meta, etc.) and CSVs into one place.
  • Unified Inventory Management: Track and (eventually) manage inventory across all channels from EEEHUB.
  • Data-Driven Insights:
    • AI-powered demand forecasting and reorder point suggestions.
    • Analysis of sales performance, inventory turnover, profit margins.
    • Identification of best/slow-selling products.
  • Streamlined Operations:
    • Automated data extraction.
    • Reduced manual data entry and reconciliation.
    • Proactive alerts for reordering and potential stock issues.
  • Improved Decision Making: Provides comprehensive analytics to help users optimize pricing, marketing, and purchasing strategies.

This architecture aims to leverage Firebase for its backend services and scalability, Python for robust data processing in Firebase Functions, and Next.js/React for a dynamic and user-friendly frontend.
